# BindingPDF

Утилита для пакетной склейки отгрузочных PDF: обходит выбранные папки, находит файлы по шаблонам имён и объединяет их в один документ по выбранному сценарию.

## Требования

- Python 3
- [PyPDF2](https://pypi.org/project/PyPDF2/) 3.x (см. `requirements.txt`)

## Установка и запуск

```bash
pip install -r requirements.txt
python BindingPDF.py
```

Пути «откуда / куда» можно один раз задать в процессе работы — они сохраняются в `config.json` рядом со скриптом.

## Общая схема (отгрузочные документы)

1. Указывается корневая папка с подпапками (обычно одна подпапка на отгрузку).
2. Задаётся **диапазон номеров** — по первому числу в **имени папки** (например, `3550-3560,3575`).
3. Выбирается тип скрепления. Итоговый файл именуется с указанием диапазона и количества комплектов.

## Сценарии скрепления

| № в меню | Описание |
|----------|----------|
| **1. Инвойсы и Спецификации** | Во всех подходящих папках ищутся `*.pdf` с `invoice` в имени. Порядок — по **номеру инвойса** в имени файла (`Invoice 123` и т.п.). |
| **2. Декларации и ЭСД** | В каждой папке — пара «одна **GTD_*.pdf**» и «один файл ЭСД» (в имени ровно **четыре дефиса**). Порядок папок — по номеру в имени папки; внутри папки берётся первая GTD и первая ЭСД по сортировке имён. |
| **3. Декларации, Инвойсы и Спецификации** | Комплект **GTD + Invoice** в каждой папке (оба `*.pdf`, GTD с префиксом `gtd_` в имени, инвойс — с `invoice` в имени). **Порядок комплектов:** по **дате** из имени GTD (средний блок из шести цифр, формат **ДДММГГ**), затем по **коду таможни** (первый числовой блок), затем по **последнему числовому блоку** (регистрационный номер). Если дата не распознана, используется только последний числовой блок, как раньше. |
| **4. Только декларации (GTD)** | В каждой папке — одна первая по сортировке имён `GTD_*.pdf`. Порядок папок — по номеру в имени папки. |

Дополнительно из главного меню:

- **Папка Temp** — склейка всех PDF из подкаталога `Temp` рядом со скриптом; порядок по числу **до первой запятой** в имени (например, `1,Doc.pdf`, `2,Doc.pdf`). Результат в `Combined`.
- **Railway** — по четыре PDF из папки `Railway`, порядок по числу в имени файла; результат в `Merged Railway`.

## Пример сортировки для сценария 3 (имена GTD)

Фрагменты вида `10137010_300426_5011010` сортируются так: сначала дата `30.04.26`, потом код таможни `10137010`, затем номер `5011010`. Сначала идут все комплекты с более ранней датой, внутри одной даты — по возрастанию кода таможни, а при совпадении таможни — по возрастанию последнего номера.
